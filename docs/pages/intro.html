Concept/Introduction

eBay-Classifieds Common api is a way to share api-functionality and design across different platform implementations. It approaches a technical interface for the classifieds domain in a Restful style. It can be seen as a common specification, which different platforms are implementing.
FILL_WITH_ARCHITECTURE_PICTURE

What is shared?

  • Domain model: For details refer to schema-definitions
  • Metadata design
  • Resource structure: Which resources (addressed by URLs) and operations (HTTP operations) on them exist.
  • Response status codes and its semantics
  • Error/Validation handling
  • Security features
  • Caching features

Concrete implementations

The concrete common-api platform implementations (see below) can decide to only implement a subset of the specification. So for all documented features please double check whether and how they are implemented (either by metadata or contacting the implementor).

Howto find out differences common-api and platform-impl

Because this documentation tells about the superset of whole "possible" common-api documentation, you have to find out which features the platform-implemenation really implements. For that refer to:
  1. If the resource and operation (addressed by URL and HTTP method) gives back 501 Not Implemented it is not supported.
  2. In case the resource and operation is supported, there is more fine grained information about supported/unsupported when accessing metadata resources . The flags 'supported/unsupported' for single-read-resource, write-single-resource and collection-resource will give you according information.
  3. Try out some sample calls. Just by looking at payload you can already gain some information (e.g. defined resolution-sizes for /pictures).
  4. If above doesn't suffice contact the platform implementor for more details (for email addresses see below).
