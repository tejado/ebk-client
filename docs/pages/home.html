Release Notes

Following release notes scope are ticket from api-client visible scope:

1.16

  • #10: Added radius info inside data-field <ad>...<address><radius>{addressRadius}<radius>...</address></ad>. This way radius info can get attached on address level (e.g. related to more detailed street + housenr. information)
  • Added fields to support contact name, contact email, visible on map <ad>... <poster-contact-email>bob@smith.com</poster-contact-email> <poster-contact-name>Bob Smith</poster-contact-name> <visible-on-map>true</visible-on-map> ...</ad>
  • #8: Added support for conversations (Message Center) (see user conversations docs, replies docs)

1.15

  • #2: Added support for auto suggestions (see docs)

1.14

  • ECG-69: Added support for User Watchlist (see docs)
  • ECG-68: Added support Search Distance, displaying distance for Ads when doing location based search (see docs)
  • ECG-66: Added support for counters (see docs)

1.13

  • ECG-59: Added support for <account-id> to ad
  • ECG-60: Added support for <username> inside reply-ad payload (see docs)
  • ECG-61: Added support for Alerts(see docs), also added device-based authentication (see Security section X-ECG-Authenticate-Device)
  • ECG-65: Add poster type option for searches

1.12

  • ECG-55: Support of dependent-attributes (see docs)

1.11

Feature:
  • ECG-49: Changing ad's state of user to PAUSED and ACTIVE (see docs)
  • ECG-52: Offer more self-references of ad for self-users self-partner (see docs)
Improvement:
  • ECG-48: Collapsing single version-schema to single one XSD, e.g. instead of ad-1.1.xsd, ad-1.3.xsd there now is only one single ad.xsd. Version is still maintained on the 'version' attribute inside XSD (see docs).

1.10

Feature:
  • ECG-39: Ability to retrieve user profile data for an authenticated user (profile, login)
  • ECG-41: Retrieve User alerts (alerts)

Feature:

  • ECG-30: Fix .json Common API JSON output format (JSON2) (see docs)
  • ECG-34: Support for <parent-id> and <children-count> optional elements for <category> and <location>

1.8

Feature:
