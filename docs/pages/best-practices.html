API Client checklists

General

If you answer following questions with 'Yes' your api-client implementation is production ready:
Did you add X-ECG-USER-AGENT header, so we can identify your current used api-client version?
Did you follow all performance tweak suggestions?
Do you evaluate all possible HTTP status responses and react respectively (e.g. recover or retry when writing to API)?
In case something goes "wrong" (e.g. 400 BadRequest or 500 server error) do you log the details of the failure see <api-base-error> tag payload? They can be valuable for later analyzation.
For your own robustness did you configure reasonable timeouts (the default most often are too long)? In case the server unexpectedly is down, has capacity or stability problems you want not your system to also go down. Watch out to tweak timeouts individually (e.g. /pictures upload will take longer as a simple GET request).
Did you inform API server implementor on expected load caused by your client implementation in production?
The representations use UTF-8 only, does your client cope (read + write) correctly with UTF-8?
In case for your platform exists a sandbox, did you use it thoroughly for testing, before switching your api-client to production API.

Read Access

For /ads Search did you check Search Summary to see whether fallbacks where done and your submitted parameter were taken?

Write Access

In case you are a partner (you want are controlling ads yourself and want to pass them to API) did you consider using /partners resource?
Does whole CRUD cycle (Create, Update, Delete, Update again) work?
Does inlining uploaded pictures (zero, one or multiple) inside ad-payload work?
Are the final locations, categories and attributes of posted ads plausible (you can check by reading ads either by view-ad or search)?
Does the final formatting of description text look alright?
